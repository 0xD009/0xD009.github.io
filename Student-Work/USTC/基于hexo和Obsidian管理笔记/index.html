# 目的

笔记是学习中很重要的工具，为了更好的利用 Obsidian 双向链接管理笔记，同时又能在 hexo 个人主页框架上发布笔记，在 github 个人仓库里发布笔记，笔记的引用格式和资源的储存方法应该得到统一，于是特此写下这篇笔记来理清思路

# 配置

# hexo

官方给的两种附件处理策略都很一般，而且我在使用 shoka 主题后渲染 md 的插件转为 hexo-renderer-multi-markdown-it ，我仔细看了两种渲染器的官方文档，修改相关配置还并不能达到使用像 Obsidian 一样的引用格式的目的

这里参考了这位师傅的博客，文中提到了 hexo-relative-link 插件，它的工作原理是在附件的路径前加一个 /../ ，这样就实现了跳回主目录的目的

# 安装方法

最后一个参数表示无视版本差异，事实上这个插件已经很久没更新里，但是它本身内容并不多，没有不兼容，所以直接安装即可

# 问题

文中所有的 md 引用格式的本地链接都会被这样改动，但超链接不会受影响，如果不想原本的链接被修改，可以通过在左括号和链接之间加空格的方式来解决

# Obsidian

# 附件管理

我们有这样一个插件

就这如它的描述，它给我们提供了 typora 一样的附件定位，可以帮助我们更好的管理附件，我个人认为还是很有必要的，后期如果需要迁移内容也方便一点

注意：它会覆盖 Obsidian 中的一些原生设置，具体指：

  1. 文件与链接中的链接表示方法，固定为相对路径表示法
  2. 文件与链接中的附件存放位置，固定为当前存放目录的特定子文件夹下

（这里建议把原来 wiki 样式的链接显示也关了，很影响笔记泛用性

# 模板管理

为了能在 Obsidian 中把博客写好，格式的要求也要注意，这里选用了 Obsidian 的第三方插件 Templater ，功能本来的核心模板插件更强，不过语法更新还挺快的

设置建议打开模板热键，有需要的话打开新建文件监听

快捷键：
Alt + E ：快速插入模板
Alt + R ：渲染变量

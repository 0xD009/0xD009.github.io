从高中就想自己捣鼓个 bot 实现类似小爱同学一样功能（那为什么不能直接用小爱同学，现在有 LLM 的 API 之后，感觉有可行性了，于是准备开干。

# 怎么把机器人跑起来

用到目前比较主流的 nonebot2 框架，以及活了超久的 gocqhttp （高中用 CoolQ 框架的时候也是它），两个都有文档站，还算比较入门友好，用初学者框架，FastAPI，onebot v11 驱动（而不是 QQ 频道）配置先创建一个项目。

这里第一个坑，不用去下载 gocqhttp，直接通过 nonebot 的脚手架安装 nonebot-plugin-gocqhttp 即可把 bot 跑起来了，甚至整了个网页控制台：

# 熟悉框架

建议通读一遍文档，了解依赖注入、装饰器和常见的 API，但是个人感觉这个文档还是不够看，还可到 github 或者 nonebot 商店看一些插件怎么写的，也有可能就直接找到你想实现的插件了，直接速通了，比如定时提醒这种。

# LLM 接入

开始尝试接入 ChatGPT，失败了，使用的插件都不行，报同一个错误，于是考虑去弄反向代理，搞了半天也是未果。

于是改投靠国内的 LLM，文心一言、ChatGLM、通义千问等，都试过，还是感觉和 GPT 差距不小，不过它们毕竟国产货，文档非常好懂。

后来偶然发现了一个逆天 bug，如果你的 urllib3 版本在 1.15.26 以上，则通过 requests 库是不能访问外网的，这正常人恐怕很难意识到。

最后成功接入了 GPT，OpenAI 的 API 文档写的也很好懂，而且传入的数据结构也差不多，没怎么改就完成了。

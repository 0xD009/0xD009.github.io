# 实验内容概述

利用所学的栈及其常用操作求解二维平面内的迷宫问题

# 实现思路

穷举，对路径进行遍历，直到找到出口

# 需要考虑的问题

  1. 避免走重复的路径
  2. 避免走出迷宫地界
  3. 遇到死路后如何实现回退
  4. 迷宫无解如何处理

# 解决办法

  1. 对路径上每一点构造方向栈，走过该方向则出栈，避免方向选择重复从而避免路径重复
  2. 进行走法合法性检验时，在原迷宫数组外围加上一圈 “栅栏”
  3. 构造路径栈，每走一步则入栈，遇到死路或方向栈为空后则出栈
  4. 检查路径栈，若为空，则说明迷宫无解

# 其他设计

  1. 实验源码在 linux 系统下编译
  2. 构造位置结构体，存有玩家目前的坐标，同时由于方向栈结构简单，为图方便，也一同放入此结构体中
  3. 程序在终端中动态执行，可以直观地看到探索迷宫的过程

# 心得体会

在接触数据结构的知识之前，我也做过类似的题，其实当初用到的方法与栈的思路完全相同，只是不知道那个叫做栈，但是系统学习了栈的知识之后，再做这个实验，我明显感觉到自己的思路比以前更加清晰，尤其将 pop() push() 两个操作封装成函数之后，不用每次都面对数组下标的问题，让整个程序显得非常清爽，可读性高

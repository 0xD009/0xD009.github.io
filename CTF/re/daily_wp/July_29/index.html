# Reverse 信安实践 Final Trial

# 1. 得到伪代码

IDA+F5，找不到主函数， SHIFT+F12 找到字符串 Input your flag: ，进入后按 X 键，再进入，按 F5 找到主函数，将伪代码整理得：

```
void __cdecl encrypt(char *a1)
{
int k; // [esp+10h] [ebp-10h]
int v2; // [esp+14h] [ebp-Ch]
int j; // [esp+18h] [ebp-8h]
int i; // [esp+1Ch] [ebp-4h]

for ( i = 0; i <= 15; ++i )
    Block[i + 256] = code1[i];
for ( j = 0; j <= 42; ++j )
    Block[j + 512] = (unsigned __int8)a1[j];
m = 0;
n = 0;
while ( 1 )
{
    switch ( code2[m] )
    {
    case 0u:
        code3[code2[m + 1]] = code3[code2[m + 2]];
        m += 3;
        break;
    case 1u:
        code3[code2[m + 1]] = *(_DWORD *)&code2[m + 2];
        m += 6;
        break;
    case 2u:
        code3[code2[m + 1]] = Block[code3[code2[m + 2]]];
        m += 3;
        break;
    case 3u:
        Block[code3[code2[m + 1]]] = code3[code2[m + 2]];
        m += 3;
        break;
    case 4u:
        code3[code2[m + 1]] ^= code3[code2[m + 2]];
        m += 3;
        break;
    case 5u:
        code3[code2[m + 1]] += code3[code2[m + 2]];
        m += 3;
        break;
    case 6u:
        code3[code2[m + 1]] -= code3[code2[m + 2]];
        m += 3;
        break;
    case 7u:
        code3[code2[m + 1]] *= code3[code2[m + 2]];
        m += 3;
        break;
    case 8u:
        code3[code2[m + 1]] %= code3[code2[m + 2]];
        m += 3;
        break;
    case 9u:
        n = code3[code2[m + 1]] - code3[code2[m + 2]];
        m += 3;
        break;
    case 10u:
        v2 = *(_DWORD *)&code2[m + 1];
        if ( !n )
        v2 = m + 5;
        m = v2;
        break;
    case 11u:
        for ( k = 0; k <= 42; ++k )
        a1[k] = Block[k + 512];
        return;
    default:
        continue;
    }
}
}

int __cdecl main(int argc, const char **argv, const char **envp)
{
char Val; // [esp+4h] [ebp-4Ch]
char Buf1[50]; // [esp+1Eh] [ebp-32h] BYREF

sub_401C40();
memset(Buf1, 0, sizeof(Buf1));
print("Input your flag: ", Val);
input("%50s", Buf1);
Block = (unsigned int *)malloc(4096u);
encrypt(Buf1);
if ( !memcmp(Buf1, changed_flag, 43u) )
    puts("Correct!");
else
    puts("Wrong, try again.");
free(Block);
return 0;
}
```

主函数逻辑简单， changed_flag 已知，主要看 encrypt()函数 。
三个 code 数组中， code3 没有定义，可以看到程序主要是在对 code2[192] 和 code3[?] 操作，尝试用脑子进行循环后发现以下特征：

  • 在除 case 10 外的分支中， m 普遍递增 3，在 case 1 中递增 6；
  • case 2 和 case 3 涉及 code3 和 Block 之间的运算，从 case 11 中可知这是此循环能加密 flag 的原因所在；
  • code 3 的值要有意义，必须经历 case 1 ， case 2 或 case 4 ;
  • code 2 的内容决定了运算方式，且是固定的；
  • Block 中的有意义值来自 flag(512-554) ， code1(256-271) ，值得重点关注。

# 2. 理清过程

  • 逆推

    • 注意到 case 11 是循环出口，查找 code2 中的 11 ，得到唯一的索引值 183 ；
    • 注意到 case 10 中涉及 m 的异常变化，查找 code2 中的 10 ，得到三个索引值 24 , 92 , 178 ，容易发现 178+5=183 ，是出循环的一种可能性；
    • m 是否 +5 与 n 的值相关， n 的值变化只在 case 9 中发生，索引值为 21 , 89 , 175 ，与 case 10 一一对应。

  • 正推
    最开始采用纸笔进行演算，效率低且出错率高，改为复制代码到 debug 环境中，利用变量监视查找规律。

    • 加密过程中除大循环外，内含 3 个小循环，对应 case 9 和 case 10 ，例如，程序第一次到 m = 21 ，进入 case 9 ，将得到 n = -255 的结果，导致在 case 10 中 m 退回为 15，每次再到 21，都会使 n + 1 ，故过程将循环 256 次，其中涉及 case 3 ，使得 Block[i] = i (0 <= i < 256) （不过知道这个其实用处不大）。
    • 第二个小循环由 41 到 92，利用了 Block 中 code1 的部分，使前 256 个数被打乱。
    • 第三个循环从 112 到 178，真正涉及到加密，前面部分完全不涉及密文，到 m = 112 之前所有过程和值都与 flag 无关。

  • 重点看第三个小循环的逻辑

    利用 Excel 逐步分析，得到大致逻辑，尝试写逆，然而，由于涉及变量太多，过程复杂， Block 在循环末的准确值难以获取，多次得到乱码后最终放弃。

# 3. 聚焦重点

尝试绕过复杂逻辑，在第三个小循环的过程中，最后产生加密效果的是变量自身对 code3[7] 取了异或运算，而 code3[7] 的值变化是由自身对 code3[2] 取异或运算，注意到 code3[2] 的值实际上与 flag 值完全无关，最终知晓得到 code3[7] 的值是解题的关键，监视 case 4 中的异或运算，打印所有 code3[7] 的值，形成容量为 43 的 key 数组，写 C 程序对 changed_flag 逐一解密，最后得到 flag ，提交，解毕。

#include<stdio.h>

int main() {
    unsigned char flag[44] = {0};

    unsigned char changed_flag[] =
    {
        139, 107, 155, 189,  56, 240, 234,  30, 101, 123, 
        100, 179,  45,  18,  56, 119,   0, 146, 125, 112, 
        251, 110, 241, 149, 253, 144, 230,  59,  48, 173, 
        194, 215, 195, 210,  46,  33, 156,  36, 243, 145, 
        247,   5, 102,   0
    };

    unsigned char key[] = 
    {
        237, 7, 250, 218, 67, 195, 188, 123, 55, 2, 
        16, 251, 65, 92, 95, 40, 82, 231, 19, 35, 
        164, 94, 159, 202, 201, 207, 171, 15, 83, 197,
        243, 185, 240, 141, 79, 25, 170, 69, 195, 242, 
        147, 51, 27
    };

    for(int i = 0; i < 43; i++) {
        flag[i] = changed_flag[i] ^ key[i];
    }
    puts(flag);

    return 0;
}

flag 为 flag{3VeRytHlNg_RunS_0n_4_M4ch1n3_a86a0cd6} .

# 4. 一点感想

这道题花的时间非常长，尤其是根据逻辑写逆求解的过程，屡改屡错，最后没有走这条路，居然用那么草率的方法成功得到了 flag，感觉像爬山爬到最险的地方又选择坐了缆车，属于是轻松没轻松到，攀上山峰的成就感也不强，嗨～

# 1. buu rsa

给了两个文件，拖进 IDA 可以看到是纯数据，加密方式自己猜，比对了前面见过的几种，都对不上，直接把 pub.key 复制进百度搜索，顺藤摸瓜找到破解方法：

  1. 公钥解析得到 e 和 n
  2. 计算得到 p 和 q
  3. 写 python 脚本
    import rsa
import gmpy2

e= 65537
n= 86934482296048119190666062003494800588905656017203025617216654058378322103517
p= 285960468890451637935629440372639283459
q= 304008741604601924494328155975272418463

phin = (p - 1) * (q - 1)
d = gmpy2.invert(e, phin)

key = rsa.PrivateKey(n,e,d,q,p)   

b = b'\x41\x96\xC0\x59\x4A\x5E\x00\x0A\x96\xB8\x78\xB6\x7C\xD7\x24\x79\x5B\x13\xA8\xF2\xCA\x54\xDA\x06\xD0\xF1\x9C\x28\xBE\x68\x9B\x62'

print(rsa.decrypt(b, key))

得到 flag{decrypt_256} ，提交，解毕。

惯用办法，一步步直接拖进去百度，发现了 ALG_ID 的存在。

找到 0x8003 和 0x8004 ：

  • CALG_MD5 0x00008003 MD5 hashing algorithm. This algorithm is supported by the Microsoft Base Cryptographic Provider.
  • CALG_SHA 0x00008004 SHA hashing algorithm. This algorithm is supported by the Microsoft Base Cryptographic Provider.

发现 MD5，用线上网站破解一下。

在 @DBAApp 前面有 12 位，结合程序要求每次输入 6 位且其中有 6 位是纯数字，依次输入 123321 , ~!3a@0 ，在程序路径下生成了一个 dbapp.rtf 文件，可以用 Word 打开，打开后就得到了 flag{N0_M0re_Free_Bugs} ，提交，正确。
还是回去看看， _strcmpi() 搜出来效果是无视大小写的字符比对，但验证后发现并没有无视，应该只是普通的字符比对， 0x8004 对应的加密方式应该这样破解：

import hashlib
flag2='@DBApp'
for i in range(100000,1000000):
     h2 = hashlib.sha1((str(i)+flag2).encode("utf-8"))
     flags = h2.hexdigest()
     if "6e32d0943418c2c33385bc35a1470250dd8923a9" == flags:
              print(str(i)+flag2)
              print(flags)

但好像把 6e32d0943418c2c33385bc35a1470250dd8923a9 直接放进 MD5 破解网站里也能得到同样结果： 123321@DBApp 。
看大佬的 wp 发现还可以通过生成文件的 sub_40100F() 入手，其实是将第二次输入的代码变换后得到指令，由文件格式可以确定变换后为 {\rtf1 ，然后逆推出变换前的六位。

# 符号还原

一般地，需要用到两个工具：
Go_Parser
Golang_Helper
来帮助 IDA 来还原符号，如果跑脚本出现报错，有可能是因为当前的脚本适配版本和 Go 版本不匹配，尤其是 Go 1.16 之后，pclntab 结构改了，很有可能出现报错，导致还原不了。

# 调用还原

还原符号后的 Go 伪码中会出现大量的 undefined var，即图中橙色的变量：

需要用 IDA 的 usercall 来还原。

一些 usercall 的函数声明例子如下：

void __usercall embed_FS_ReadFile(_QWORD@<rax>, _QWORD@<rbx>, _QWORD@<rcx>)
_QWORD __usercall fmt_Fprintln@<rax>(const void *@<rax>, void *@<rbx>, void *@<rcx>, _QWORD@<rdx>, int@<rdi>, int@<rsi>)
struct GoString {  
  const char* data;  
  const char* length;  
};  
GoString __usercall main_randomDinosaur@<rbx:rax>()
GoString *__usercall runtime_convTstring@<rax>(_QWORD@<rax>, _QWORD@<rbx>)

Golang 中传参和返回值常用的寄存器：rax, rbx, rcx, rdx (不确定有没有), rdi, rsi。

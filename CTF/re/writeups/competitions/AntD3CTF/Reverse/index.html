# d3syscall

在 init_array 中加载了名为 /tmp/mymodule 的 syscall_table ，实现了一种很新的 vm

指令表
0x14F avg1 avg2 avg3 赋值语句
avg1 == 0: data [avg2] = data [avg3]
avg1 == 1: data [avg2] = avg3

0x150 avg1 avg2 avg3 计算语句
avg1 == 0: data [avg2] += data [avg3]
avg1 == 1: data [avg2] -= data [avg3]
avg1 == 2: data [avg2] *= data [avg3]
avg1 == 3: data [avg2] ^= data [avg3]
avg1 == 4: data [avg2] <<= data [avg3]
avg1 == 5: data [avg2] >>= data [avg3]

0x151 avg1 avg2 入栈语句
avg1 == 0: push (data [avg2])
avg1 == 1: push (avg2)

0x152 avg1 出栈语句
pop (data [avg1])

0x153 清理语句
clear (data [0:4])

加密过程整理后利用 z3 求解，z3 约束反映了对加密过程的理解，所以不再赘述加密过程

# d3recover

本题基于 Cython，给了两个附件，利用 Bindiff 将 2.0 的符号导入 1.0 中方便分析，加密逻辑为：对 32 字节的输入，先异或 0x23，其次，第 i 字节与第 i + 2 字节相加按位与 255，再异或 0x54，最后 base64encode
